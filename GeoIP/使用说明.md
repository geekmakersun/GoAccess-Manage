# GeoIP 数据库更新脚本使用说明

## 📋 脚本信息

- **脚本位置**: `GeoIP/更新GeoLite2.sh`
- **适用环境**: Linux/Unix 系统（CentOS/Ubuntu/Debian 等）
- **Windows 用户**: 需要在 Linux 服务器上运行，或使用 WSL/Git Bash

## 🎯 功能特性

### ✅ 版本号持久化
- 创建 `GeoIP.version` 文件记录数据库版本信息
- 包含版本、日期、大小、更新时间等信息
- 自动更新，无需手动维护

### ✅ 智能版本检测
- 自动检测数据库年龄
- 超过 30 天才执行更新
- 避免不必要的重复下载

### ✅ 多镜像源支持
- GitHub（主源）
- jsDelivr CDN（备用）
- Fastly CDN（备用）
- 自动切换，提高成功率

### ✅ 原子更新策略
- 先下载到临时文件
- 验证文件大小（> 1MB）
- 成功后再替换原文件
- 自动备份旧版本

### ✅ 自动清理机制
- 自动清理超过 7 天的备份文件
- 避免备份文件占用过多空间

## 📖 使用方法

### Linux 服务器上运行

```bash
# 进入 GeoIP 目录
cd /www/wwwroot/GoAccess-管理/GeoIP

# 更新所有数据库（自动检测版本）
./更新GeoLite2.sh

# 只更新 City 数据库
./更新GeoLite2.sh -c

# 只更新 ASN 数据库
./更新GeoLite2.sh -a

# 强制更新（忽略版本检查）
./更新GeoLite2.sh -f

# 显示版本信息
./更新GeoLite2.sh -v

# 清理旧的备份文件
./更新GeoLite2.sh -C
```

### Windows 用户

#### 方式 1: 在 Linux 服务器上运行（推荐）
将脚本上传到 Linux 服务器后执行

#### 方式 2: 使用 WSL
```bash
# 安装 WSL
wsl --install

# 在 WSL 中运行
cd /mnt/c/Users/sunbingchen/工作区/GoAccess-管理/GeoIP
./更新GeoLite2.sh
```

#### 方式 3: 使用 Git Bash
```bash
# 在 Git Bash 中运行
cd /c/Users/sunbingchen/工作区/GoAccess-管理/GeoIP
./更新GeoLite2.sh
```

## 🔄 工作流程

1. **检测操作系统** - 自动识别运行环境
2. **显示当前版本** - 启动时显示已安装的数据库版本
3. **检查版本年龄** - 超过 30 天才执行更新
4. **下载最新版本** - 从多个镜像源下载
5. **验证文件大小** - 确保下载完整（> 1MB）
6. **原子更新** - 先下载到临时文件，验证后替换
7. **自动备份** - 备份旧版本到带时间戳的文件
8. **更新版本文件** - 保存最新的版本信息到 `GeoIP.version`
9. **显示最新版本** - 更新完成后显示新版本信息
10. **清理备份** - 自动清理超过 7 天的备份文件

## ⚙️ 定时任务设置

### 宝塔面板定时任务

在宝塔面板添加计划任务：

```bash
# 每周二凌晨 3 点自动更新（MaxMind 每周二更新数据库）
0 3 * * 2 cd /www/wwwroot/GoAccess-管理/GeoIP && ./更新GeoLite2.sh >> /var/log/geoip-update.log 2>&1
```

### Linux Crontab

```bash
# 编辑 crontab
crontab -e

# 添加定时任务（每周二凌晨 3 点）
0 3 * * 2 cd /www/wwwroot/GoAccess-管理/GeoIP && ./更新GeoLite2.sh
```

## 📝 注意事项

1. **运行环境**: 脚本需要在 Linux/Unix 系统上运行，Windows 用户请使用 WSL 或 Git Bash
2. **权限要求**: 需要有写入 GeoIP 目录的权限
3. **磁盘空间**: 至少需要 100MB 可用空间
4. **网络连接**: 需要能够访问 GitHub 和 CDN
5. **版本检测**: 数据库超过 30 天才会更新，避免频繁下载
6. **自动备份**: 更新前会自动备份，备份文件保留 7 天

## 🔧 故障排除

### 问题 1: 无法运行脚本
**原因**: Windows PowerShell 不支持 bash 命令
**解决**: 在 Linux 服务器上运行，或使用 WSL/Git Bash

### 问题 2: 下载失败
**原因**: 网络问题或镜像源不可用
**解决**: 脚本会自动切换镜像源，或手动下载数据库文件

### 问题 3: 权限不足
**原因**: 没有写入权限
**解决**: 使用 root 用户或 sudo 运行脚本

### 问题 4: 磁盘空间不足
**原因**: 可用空间小于 100MB
**解决**: 清理不需要的文件，释放磁盘空间
